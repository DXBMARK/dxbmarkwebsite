# Webhook Interface: HTTP Route & Signature Validation Contract

This document defines the HTTP API endpoint contract for incoming Stripe webhook events.

---

## 1. HTTP Endpoint Specification

- **Method**: `POST`
- **Path**: `/api/stripe/webhook`
- **Content-Type**: `application/json`
- **Headers**:
  - `stripe-signature`: Required. Cryptographic signature generated by Stripe to verify payload integrity.

---

## 2. Request Verification Protocol

The Next.js App Router API endpoint must verify payload integrity using the following sequence:

1. **Extract Raw Body**:
   - The handler MUST retrieve the body as a raw string using `request.text()`.
   - Never parse the body as JSON prior to signature verification.
2. **Retrieve Signature Header**:
   - Read the `stripe-signature` header from request headers.
   - If missing, return `400 Bad Request` immediately.
3. **Verify Cryptographic Signature**:
   - Invoke `stripe.webhooks.constructEvent(rawBody, signature, webhookSecret)`.
   - If verification throws an exception, return `400 Bad Request` immediately.
4. **Enforce Time Tolerance**:
   - Validate that the timestamp in the signature header matches Stripe's configured time tolerance window (default 300 seconds) to prevent replay attacks.

---

## 3. Response Codes

- **`200 OK`**:
  - Returned when the event is successfully verified, checked for uniqueness, and logged in the event store.
  - Also returned for ignored event types or duplicate events to avoid blocking Stripe retry queues.
- **`400 Bad Request`**:
  - Returned when signature verification fails, the signature header is missing, or the payload format is invalid.
- **`500 Internal Server Error`**:
  - Returned only when critical database errors occur prior to raw response dispatch. Sentry alerts are triggered immediately.
